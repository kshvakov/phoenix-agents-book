# Автономные агенты: как трансформировать инженерию и бизнес

**В стиле \"The Phoenix Project\" — от хаоса к эффективности через ИИ-агентов**

**Целевая аудитория:**
- **Основная:** Senior/Staff/Principal+ инженеры с опытом разработки в продакшене
- **Вторичная:** руководители разработки (EM/VP Engineering), CTO
- **Третья:** продакт‑ и деливери‑менеджеры, бизнес‑стейкхолдеры

**Это не учебник по промптам.** Это книга о трансформации инженерной культуры через автономных агентов, где технические детали (промпты, `SOP`, контрольные точки) служат достижению измеримых бизнес‑результатов.

> **Статус:** Work in Progress. Главы 1–10 в активной разработке, структура стабилизировалась.

---

## Как читать эту книгу

**Быстрый старт для инженеров:**
1. Начните с [Главы 1]({{< relref "chapters/01-first-prompt.md" >}}) — сделайте первый рабочий промпт и план проверки
2. Пройдите “Быстрый старт” в каждой главе для немедленной практической пользы
3. Возвращайтесь к полному материалу главы по мере необходимости

**Для менеджеров и бизнеса:**
- Начните с краткого резюме в начале каждой главы
- Изучите [Приложение A: Бизнес‑кейс]({{< relref "appendix/A-business-case.md" >}}) как шаблон для бизнес‑обоснования
- Изучите [Приложение B: Организационная трансформация]({{< relref "appendix/B-organizational-transformation.md" >}}) как плейбук управления изменениями
